About Swiggy & Group Orders

The Platform

Swiggy is one of India's leading online food delivery and quick-commerce platforms. It connects users with thousands of restaurants, cloud kitchens, and grocery stores.

The app provides several services such as Food delivery, Instamart for groceries, Dineout deals, and Pick-up & drop services.

Group Order Feature

Allows multiple people to add items to a single shared order. Designed for friends, colleagues, roommates, or family members wanting to order food together.

How it works: Create group -> Generate shareable link -> Members join -> Everyone adds items -> Host pays & places order.

Project Brief

This project focuses on enhancing the Group Order feature to improve convenience and efficiency. The proposed solution introduces a "Create & Save Groups" option, allowing users to form reusable groups (e.g., office team, roommates) for faster ordering. It also integrates a secondary split payment feature so participants can pay for individual items after the order is placed, reducing the burden on a single payer.

Problem & Opportunity

⚠️

The Problem

The current experience forces users to choose a restaurant first before inviting others. This creates friction because:

  • Not everyone wants food from the same restaurant
  • People discuss options in unorganized chat apps
  • The host must coordinate orders manually
Example Situation:
Rahul → Wants Pizza Priya → Wants Biryani

But the current system only allows ONE restaurant selection.

💡

The Opportunity & Goal

Allow users to create the group first, then let members "choose their own restaurants" while delivering to the "same location".

System Goals:

  • The host creates a group
  • Members join using a link
  • Everyone orders from any restaurant
  • All deliveries arrive at the same location

The Split Payment Dilemma

Our research revealed that coordinating the financial aspect of group orders is often more stressful than agreeing on what to eat.

By introducing a native split payment feature and allowing independent carts, we remove the awkwardness of chasing friends for money and integrate everything into one seamless flow.

Multi-Restaurant Flexibility

Why should everyone compromise on a single cuisine?

The new flow lets users build a shared cart but pick from entirely different restaurants. All orders arrive at the same destination, keeping the group together without sacrificing choice.

UserJourney Map

Swiggy Group Order Enhancement - Mapping the user actions, pain points, and design solutions.

Stage User Actions Thoughts / Needs Pain Points Opportunities (Design Solutions)
1. Discover / Need User wants to order food with friends, roommates, or colleagues. "Everyone wants something different. How do I collect all orders quickly?" Taking orders in WhatsApp or manually adding items is confusing. Provide Group Order entry point on restaurant page with clear CTA.
2. Create / Select Group User starts a group order and invites people. "I order with the same people often." Re-adding contacts every time is repetitive and time-consuming. Introduce Saved Groups (Office Team, Flatmates, Friends). Users can create or reuse groups.
3. Add Members & Share Link User shares invite link or selects saved group members. "Everyone should add their own food." Some members forget to add items or miss the link. Show live member activity, reminders, and join notifications.
4. Add Items to Cart Each participant browses the menu and adds their own items. "I only want to pay for my food." One person usually ends up paying the full bill. Add Secondary Split Payment option so each member pays their share.
5. Review & Checkout Host reviews cart and places order. "Is everyone done adding items?" Confusion about order completion and payment responsibility. Show member order summary + confirmation before checkout.
6. Payment & Split Host pays initially or selects split payment option. "I shouldn't chase everyone for money later." Collecting money through UPI after ordering is inconvenient. Enable auto payment request / split bill inside the app.
7. Delivery & Post-Order Group tracks delivery and receives food together. "That was easier than before." No easy way to repeat same group order later. Add Reorder with Saved Group for faster future orders.
